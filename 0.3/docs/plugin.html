缘由

所谓接口测试，就是给出请求数据，然后对比响应数据。

有时我们无法直接给出请求数据, 比如当前时间。 有时我们也无法完全确定响应数据，比如返回值可能尾部跟个随机字段。测试具有灵活性，而 YAML 一般只能提供具体值。我们无法给出随时在变的“当前时间”。

我们需要函数，只有函数才具有灵活性，才能应付这些挑战。

插件正是为了提供这些函数。

原理

为了理解插件工作原理，我们必须先学习一些 YAML 规范。

在 YAML 中，有 4 种数据类型。

  • 标量(scalar)
  • 数组(sequence)
  • 散列表(mapping)
  • 标签(tag)

其中标签(tag)是实际上是一种函数。YAML 解析引擎碰到标签后，会查找标签库中是否存在该标签，如果存在就获取该标签绑定的函数，将标签后的数据作为参数传递给标签函数，以函数返回值作为最终数据。

插件将函数封装成 YAML 标签，Htte 加载插件并将这些标签注册到 YAML 解析引擎中。函数在 YAML 中以 YAML 标签的形式存在。

种类

htte 提供了两类插件

  • resolver
    用来生成请求数据, 其 yaml 标签带有前缀 !$, 如 !$datetime 返回时间字符串

  • differ
    用来对比验证响应数据, 其 yaml 标签带有前缀 !@, 如 !@regexp 验证数据是否是否匹配正则

内置插件

Htte 提供了一些内置插件，这些插件不需要额外安装，可以直接使用。

这些插件的功能和基础，一般测试中都有用到。 它们也是例子，可以参照它们编写你自己的插件。如果你发现某个重要的功能应该包含在内置插件中，欢迎提交 issue 或 pull request.

!$query

!$query: 获取数据链路指向的数据, 参数类型为 scalar

!$concat

!$concat: 连接一组字符串, 参数类型为 sequence

!$datetime

!$datetime: 生成时间字符串, 参数类型为 sequence

接受一个数组，第一个参数是控制选项 <offset>:<format> , 第二个是基准时间

偏移量是时间字符串参考 timestring
支持的格式参考 dateformat

!$randstr

!randstr: 随机字符串，参数类型为 scalar

接受一个参数 <length>:<flag> 用来控制随机数生成。 flag 有三个基本类: l 表示小写, u 表示大写，n 表示数字，flag 间可以组合

!@query

!@query: 比对数据是否与链路指向数据相等, 参数类型为 scalar

如果响应数据有字段 email 且与请求数据 email 值相等，为真

!@exist

!@exist: 是否存在字段，参数类型为 scalar

如果测试返回的结果是对象，有且仅有属性 username, token, 为真。不关注具体值，仅关注属性存在与否。

!@regexp

!@regexp: 是否匹配正则，参数类型为 scalar

如果响应数据有字段 slug，为字符串且匹配正则，为真

!@array

!@array: 数组部分校验，参数类型为 mapping

如果响应数据字段值为数组且第一个元素值为 dragon，为真
这里我们仅对第一个元素敢兴趣，所以仅做部分校验。

如果我们还对数组长度敢兴趣，可以断言长度

Htte 默认情况下会对数组进行全等校验。这意味着必须列出全部元素，并保证顺序和值正确。

!@object

!@object: 对象部分校验，参数类型会 mapping

如果响应数据中用字段 email 且值为 john@jacob 为真。如果省略 !@object 标签，表示数据有且仅有字段 email。

Htte 默认情况下会对对象进行全等校验。不能有多的字段，也不能有少的字段。
