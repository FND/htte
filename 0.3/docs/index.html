欢迎使用 Htte 文档。文档详细介绍了 Htte 的方方面面，建议使用 Htte 前读一读文档。如果你在使用 Htte 过程中碰到了任何问题，欢迎提交 issue。

定义

Htte 是一款描述式 HTTP 接口自动测试框架。它允许你使用 YAML 编写 HTTP 接口测试。Htte 也是一款发布在 npm 上的命令行工具。该工具执行编写的测试 YAML 文件并打印测试结果。　

特性

  • 自动化，逐一执行测试文件中的测试用例
  • 声明式，用 YAML 编写测试
  • 语言无关性，平台无关性
  • 插件化，可以自由生成响应数据，灵活校验响应数据
  • 编码可扩展，内置 JSON/XML 编码器
要求

Node.js 最低版本 v4.x.x。

竞品

vs Postman

相同点:

  • 后端语言不敏感
  • 自动执行测试单元并报告结果

不同点:

  • Postman 执行 json 文件，Htte 执行 YAML 文件，YAML 更富有表现力。
  • Postman 必须采用其特制 IDE 编写，Htte 可以使用任何文本编辑器，Htte 编写测试更灵活更高效。
  • Postman 通过环境变量应用其它测试的数据(Authorization 头引用登录返回 token)，Htte 使用数据链路，Htte 数据引用策略更安全合理便捷。
  • Postman 使用 JS 断言响应数据，Htte 直接根据 YAML 描述的响应信息与实际返回的响应信息比对以此断言响应数据，Htte 更简便。
