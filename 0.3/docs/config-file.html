配置

配置可以用来控制 Htte 行为。这些配置信息统一存储到一个 yaml 格式的配置文件中。其位置需要使用命令行参数 -c 或 --config 指定

默认值

各配置项的默认值:

配置项

rootDir

指定测试文件所在的根目录。

Htte 会递归读取该目录下的所有 yaml 文件以解析加载测试用例。

sessionFile

会话持久化文件路径。

Htte 使用该文件记录测试单元的请求响应以及测试中断位置。

会话表示 Htte 一次执行过程。会话持久化使 Htte 具有从特定位置执行测试的能力，而不用每次测试都需要从头执行所有测试。我们可以针对某个异常接口进行“测试－失败－修改-再测试”,直到它成功。这让我们可以采用 TDD 开发接口。

apis

描述项目中使用的接口。

接口属性有:

  • name: 接口名字。
  • uri: 接口访问路径，可以是相对路径如 /users/login, 也可以是绝对路径 http://localhost:3000/users/login。
  • method: 接口请求方法，可选值: get, head, post, put, delete, connect, options, patch。
  • type: 接口请求数据类型，Htte 根据该值添加请求头 Content-Type，编码请求数据到请求体。
  • timeout: 接口请求超时，请求超过该时间会自动中断并报错。

url

接口访问路径基地址。

接口对象中的 uri 是相对地址时，自动与该基地址拼接得到绝对地址。

接口anything 的完整地址时 https://httpbin.org/anything。

type

接口默认请求数据格式。

为接口属性 type 提供缺省值。

timeut

接口默认请求超时时间, 单位为毫秒。

为接口属性 timeut 提供缺省值。
